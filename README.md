# MenuOS

The ultimate QR Menu SaaS platform. 

## Getting Started

First, install the dependencies using pnpm:

```bash
pnpm install
```

Run the development server:

```bash
pnpm dev
```

Open [http://localhost:3000](http://localhost:3000) with your browser to see the result.

## Cloudflare Pages Deployment

This project uses `@cloudflare/next-on-pages` for edge deployment. Do not deploy this to Vercel or standard Node.js environments.

1. Install the Cloudflare CLI `wrangler`: `npm install -g wrangler`
2. Run a local build to test: `pnpm run pages:build`
3. Connect your GitHub repository to Cloudflare Pages.
4. Set the build command to: `npx @cloudflare/next-on-pages`
5. Set the output directory to: `.vercel/output/static`
6. Add the Node.js version compatibility flag: Add environment variable `NODE_VERSION` = `20` (or greater).
7. Ensure all Supabase and CDN environment variables are provided in the Cloudflare Pages settings.

## Custom Domain Setup Guide

If you're on the **Premium** tier, follow these instructions to set up a custom domain (e.g., `menu.yourrestaurant.com`) for your MenuOS deployment:

1. **Log in to Cloudflare Dashboard**:
   Navigate to the Cloudflare dashboard and select your `menuos` Pages project.
2. **Go to Custom Domains**:
   Click on the **Custom Domains** tab within your project settings.
3. **Set Up Domain**:
   Click **Set up a custom domain**. Enter the full domain or subdomain you want to use (e.g., `menu.yourrestaurant.com`).
4. **DNS Configuration**:
   - *If your domain's DNS is already managed by Cloudflare*: Cloudflare will automatically add the required CNAME record for you. Simply click **Activate**.
   - *If your domain is managed elsewhere (e.g., GoDaddy, Namecheap)*: Cloudflare will provide a CNAME record (usually pointing to `yourproject.pages.dev`). Log in to your domain registrar, navigate to the DNS settings, and add the CNAME record as instructed.
5. **Wait for Validation**:
   It may take up to 48 hours for DNS changes to propagate globally, but it typically happens within 5-10 minutes. Once Cloudflare verifies the DNS records, it will automatically provision an SSL certificate and activate your custom domain!

*Note: Your `NEXT_PUBLIC_SITE_URL` environment variable will still dictate absolute paths generated by the system (such as QR codes and SEO metadata), so remember to update it to your new custom domain once activated.*
